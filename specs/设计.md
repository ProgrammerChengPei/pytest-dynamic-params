# pytest 动态参数化系统设计文档

## 1. 系统概述

### 1.1 问题背景
在现有的 pytest 测试框架中，当需要结合静态参数（通过 `@pytest.mark.parametrize` 定义）和动态生成的参数时，会遇到以下核心问题：

1. **参数化时机冲突**：`@pytest.mark.parametrize` 在**模块导入阶段**执行，而 fixture 在**测试运行阶段**才实例化
2. **无法访问运行时数据**：在装饰器参数中无法直接调用依赖 fixture 的函数
3. **代码结构复杂**：需要在钩子中手动处理参数收集、组合和动态生成逻辑
4. **缺乏依赖管理**：无法自动处理参数间的依赖关系，需要手动排序

### 1.2 设计目标
1. **声明式参数生成**：通过装饰器声明参数生成逻辑
2. **自动依赖解析**：自动分析参数依赖关系并按正确顺序执行
3. **高性能参数生成**：支持懒加载、增量生成和智能缓存
4. **与现有机制兼容**：与 `@pytest.mark.parametrize` 和 fixture 系统无缝集成
5. **良好的错误处理**：提供清晰的错误提示，帮助用户快速定位问题

### 1.3 核心概念
- **参数生成器**：定义如何生成参数值的函数
- **动态参数**：在运行时生成的参数，可以依赖其他参数或 fixture
- **参数依赖**：参数之间的依赖关系，系统需要按依赖顺序生成参数
- **懒加载**：推迟参数生成到实际需要时
- **增量生成**：当参数变化时，只重新生成受影响的部分

## 2. 功能架构设计

### 2.1 整体架构
```
用户层
  ├── 测试用例 (使用@with_dynamic_params)
  └── 参数生成器 (使用@param_generator)
         ↓
接口层
  ├── 装饰器接口 (@param_generator, @with_dynamic_params)
  └── 配置接口 (配置文件、环境变量)
         ↓
核心层
  ├── 参数收集器 (ParamCollector)
  ├── 生成器注册表 (GeneratorRegistry)
  ├── 依赖解析器 (DependencyResolver)
  ├── 组合生成器 (CombinationGenerator)
  ├── 参数一致性验证器 (ParamConsistencyValidator)
  └── 错误处理器 (ErrorHandler)
         ↓
集成层
  ├── pytest_generate_tests钩子集成
  ├── fixture系统集成
  └── 缓存系统集成
```

### 2.2 核心组件职责

#### 2.2.1 参数生成器注册表 (GeneratorRegistry)
**职责**：
- 注册和管理所有参数生成器
- 提供生成器查找功能
- 维护参数名到生成器的映射关系

**关键属性**：
- `_generators: Dict[str, ParamGenerator]` - 生成器映射
- `_param_to_generator: Dict[str, str]` - 参数名到生成器名的映射

**关键方法**：
- `register(func: Callable) -> Callable` - 注册生成器
- `get_generator(param_name: str) -> Optional[ParamGenerator]` - 获取参数生成器
- `get_all_generators() -> List[ParamGenerator]` - 获取所有生成器

#### 2.2.2 依赖解析器 (DependencyResolver)
**职责**：
- 分析参数生成器的依赖关系
- 检测循环依赖
- 确定生成器的执行顺序

**关键方法**：
- `resolve_order(generators: List[ParamGenerator]) -> List[ParamGenerator]` - 解析执行顺序
- `detect_cycles() -> List[List[str]]` - 检测循环依赖
- `get_dependency_graph() -> Dict[str, List[str]]` - 获取依赖图

#### 2.2.3 参数收集器 (ParamCollector)
**职责**：
- 收集测试用例中的静态参数
- 收集 fixture 信息
- 识别需要动态生成的参数

**关键方法**：
- `collect_static_params(metafunc) -> Dict[str, List[Any]]` - 收集静态参数
- `collect_fixture_info(metafunc) -> Dict[str, Any]` - 收集 fixture 信息
- `identify_dynamic_params(metafunc) -> List[str]` - 识别动态参数

#### 2.2.4 组合生成器 (CombinationGenerator)
**职责**：
- 生成所有可能的参数组合
- 支持懒加载参数组合
- 处理参数值的交叉组合

**关键方法**：
- `generate_combinations(param_dict: Dict[str, List[Any]]) -> List[Dict[str, Any]]` - 生成参数组合
- `generate_lazy_combinations(static_params, lazy_generators) -> List[Dict[str, Any]]` - 生成懒加载组合
- `filter_combinations(combinations, filters) -> List[Dict[str, Any]]` - 过滤组合

#### 2.2.5 参数一致性验证器 (ParamConsistencyValidator)
**职责**：
- 验证参数名一致性
- 检查参数依赖的可用性
- 提供参数验证报告

**关键方法**：
- `validate_param_mapping(param_mapping, available_params) -> ValidationResult` - 验证参数映射
- `check_dependencies(generator, available_params) -> List[str]` - 检查依赖可用性
- `generate_validation_report() -> str` - 生成验证报告

### 2.3 基本交互流程
1. **测试用例定义**：用户使用 `@with_dynamic_params` 装饰器标记测试用例
2. **参数收集**：在 `pytest_generate_tests` 钩子中收集所有可用参数
3. **依赖解析**：解析动态参数的依赖关系
4. **参数组合生成**：生成所有参数组合
5. **动态参数生成**：按顺序执行参数生成器
6. **测试参数化**：调用 `metafunc.parametrize` 进行参数化
7. **测试执行**：pytest 执行参数化的测试用例

## 3. 性能架构设计

### 3.1 性能优化目标
1. **减少不必要的参数生成**：通过懒加载推迟参数生成时机
2. **重用已生成的参数**：通过缓存避免重复生成
3. **按需生成参数值**：通过增量生成只重新生成变化的部分

### 3.2 懒加载机制设计

#### 3.2.1 懒加载执行流程
```
1. 测试收集阶段：
   ├── 识别需要懒加载的参数
   ├── 创建懒加载生成器对象
   ├── 生成参数组合结构（包含懒加载占位符）
   └── 注册到测试参数化

2. 测试执行阶段（fixture调用时）：
   ├── 收集执行上下文
   ├── 检查缓存
   ├── 如果未缓存，执行生成器
   ├── 更新缓存
   └── 返回结果
```

#### 3.2.2 懒加载生成器 (LazyGenerator)
**职责**：
- 封装生成器函数的懒加载逻辑
- 管理执行状态和缓存
- 提供统一的接口供 fixture 调用

**关键属性**：
- `generator_func: Callable` - 实际的生成器函数
- `param_name: str` - 生成的参数名
- `_executed: bool` - 是否已执行
- `_result: Optional[Any]` - 执行结果
- `_context: Optional[Dict]` - 执行上下文

**关键方法**：
- `set_context(context: Dict)` - 设置执行上下文
- `get_value() -> Any` - 获取值（懒加载执行）
- `execute() -> Any` - 执行生成器

#### 3.2.3 懒加载与核心组件的交互
1. **与 ParamCollector 的交互**：
   - ParamCollector 识别需要懒加载的参数
   - 为懒加载参数创建 LazyMarker

2. **与 CombinationGenerator 的交互**：
   - CombinationGenerator 生成包含 LazyMarker 的组合
   - 组合中包含懒加载参数的占位符

3. **与 fixture 系统的交互**：
   - 创建懒加载 fixture
   - 在 fixture 中执行懒加载生成器

### 3.3 增量生成机制设计

#### 3.3.1 增量生成触发条件
- **参数值变化**：静态参数或 fixture 的值发生变化
- **依赖关系变化**：参数生成器的依赖关系发生变化
- **生成器变化**：参数生成器函数发生变化

#### 3.3.2 变化检测器 (ChangeDetector)
**职责**：
- 检测参数变化
- 记录参数状态
- 识别变化的参数

**关键属性**：
- `_previous_states: Dict[str, Any]` - 历史状态记录
- `_dependency_graph: Dict[str, List[str]]` - 依赖关系图
- `_change_history: Dict[str, List[ChangeRecord]]` - 变化历史

**关键方法**：
- `detect_changes(current_state: Dict[str, Any], item_id: str) -> List[str]` - 检测变化
- `record_state(state: Dict[str, Any], item_id: str)` - 记录状态
- `get_changed_params(item_id: str) -> List[str]` - 获取变化的参数

#### 3.3.3 增量生成协调器 (IncrementalGenerationCoordinator)
**职责**：
- 协调增量生成过程
- 管理受影响参数链
- 协调缓存更新

**关键属性**：
- `dependency_resolver: DependencyResolver` - 依赖解析器
- `cache_manager: CacheManager` - 缓存管理器
- `change_detector: ChangeDetector` - 变化检测器

**关键方法**：
- `generate_incrementally(test_item, param_generators, static_params) -> Dict[str, Any]` - 增量生成参数
- `_get_affected_generators(changed_params, all_generators) -> Set[ParamGenerator]` - 获取受影响的生成器
- `_regenerate_affected(generators, context) -> Dict[str, Any]` - 重新生成受影响的部分

#### 3.3.4 增量生成与核心组件的交互
1. **与依赖解析器的交互**：
   - 获取完整的依赖关系图
   - 分析受影响的参数链

2. **与缓存管理器的交互**：
   - 查询未受影响的缓存结果
   - 增量更新缓存

3. **与生成器执行器的交互**：
   - 只执行受影响的生成器
   - 重用未受影响的生成结果

### 3.4 缓存策略设计

#### 3.4.1 多级缓存架构
```
1. 内存缓存 (MemoryCache)
   ├── 立即值缓存
   └── 懒加载值缓存

2. 磁盘缓存 (DiskCache) - 可选
   ├── 序列化缓存结果
   └── 支持会话间缓存

3. 作用域缓存 (ScopeCache)
   ├── function级别缓存
   ├── class级别缓存
   ├── module级别缓存
   └── session级别缓存
```

#### 3.4.2 缓存键生成策略
- **参数哈希**：基于参数值的哈希
- **依赖签名**：基于依赖关系的签名
- **作用域标识**：基于作用域的标识符
- **生成器标识**：生成器函数的唯一标识

### 4.2 关键算法设计

#### 4.2.1 依赖解析算法（拓扑排序）
```
算法: 依赖解析与排序
输入: generators - ParamGenerator列表
输出: ordered_generators - 按依赖顺序排序的生成器列表
异常: CircularDependencyError - 检测到循环依赖时抛出

伪代码:
function resolve_order(generators):
    # 构建依赖图
    graph = {}
    for generator in generators:
        deps = []
        for dep in generator.dependencies:
            # 找到依赖的生成器
            for other in generators:
                if other.param_name == dep:
                    deps.append(other.param_name)
                    break
        graph[generator.param_name] = deps
    
    # 计算入度
    in_degree = {param_name: 0 for param_name in graph}
    for deps in graph.values():
        for dep in deps:
            in_degree[dep] += 1
    
    # 拓扑排序（Kahn算法）
    queue = deque([param_name for param_name, deg in in_degree.items() if deg == 0])
    result = []
    
    while queue not empty:
        param_name = queue.popleft()
        result.append(get_generator_by_param_name(param_name))
        
        for dep in graph[param_name]:
            in_degree[dep] -= 1
            if in_degree[dep] == 0:
                queue.append(dep)
    
    # 检查是否存在循环依赖
    if len(result) != len(generators):
        # 找到剩余的节点，它们形成循环
        remaining = [param_name for param_name in graph if param_name not in result]
        raise CircularDependencyError(remaining)
    
    return result
```

#### 4.2.2 懒加载组合生成算法
```
算法: 懒加载参数组合生成
输入: 
  - static_params: Dict[str, List[Any]] - 静态参数及其值列表
  - lazy_generators: List[ParamGenerator] - 懒加载生成器列表
输出: List[Dict[str, Any]] - 包含LazyResult的参数组合

伪代码:
function generate_lazy_combinations(static_params, lazy_generators):
    # 分离懒加载参数和立即参数
    lazy_param_names = [g.param_name for g in lazy_generators]
    
    # 生成立即参数的所有组合
    immediate_combinations = []
    if static_params:
        param_names = list(static_params.keys())
        param_values = list(static_params.values())
        
        for combo in itertools.product(*param_values):
            combination = dict(zip(param_names, combo))
            immediate_combinations.append(combination)
    else:
        immediate_combinations = [{}]
    
    # 为每个立即组合创建懒加载生成器
    all_combinations = []
    for combo in immediate_combinations:
        new_combo = combo.copy()
        
        for generator in lazy_generators:
            # 创建懒加载结果
            lazy_result = LazyResult(
                generator=generator,
                context=combo,  # 使用立即参数作为初始上下文
                cache_key=generator._make_cache_key(combo)
            )
            new_combo[generator.param_name] = lazy_result
        
        all_combinations.append(new_combo)
    
    return all_combinations
```

#### 4.2.3 增量生成变化检测算法
```
算法: 增量生成变化检测
输入:
  - previous_state: Dict[str, Any] - 前一次参数状态
  - current_state: Dict[str, Any] - 当前参数状态
  - dependency_graph: Dict[str, List[str]] - 依赖关系图
输出: List[str] - 受影响的参数列表（需要重新生成的参数）

伪代码:
function detect_affected_parameters(previous_state, current_state, dependency_graph):
    # 1. 检测变化的参数
    changed_params = []
    for param_name, current_value in current_state.items():
        previous_value = previous_state.get(param_name)
        if not _deep_equals(previous_value, current_value):
            changed_params.append(param_name)
    
    # 2. 通过依赖图找到所有受影响的参数
    affected = set(changed_params)
    queue = deque(changed_params)
    
    while queue not empty:
        param = queue.popleft()
        
        # 找到直接依赖param的参数
        for dependent_param, deps in dependency_graph.items():
            if param in deps and dependent_param not in affected:
                affected.add(dependent_param)
                queue.append(dependent_param)
    
    return list(affected)
```

```markdown
   └─────────────────────────────────────────────────────┘

2. 懒加载执行阶段 (fixture调用时):
   ┌─────────────────────────────────────────────────────┐
   │ 1. 获取懒加载结果对象                               │
   │ 2. 检查缓存                                         │
   │ 3. 执行生成器（如果需要）                           │
   │ 4. 返回结果                                         │
   └─────────────────────────────────────────────────────┘

3. 增量生成流程:
   ┌─────────────────────────────────────────────────────┐
   │ 1. 检测参数变化                                     │
   │ 2. 解析受影响的参数链                               │
   │ 3. 重新生成受影响的部分                             │
   │ 4. 更新缓存                                         │
   └─────────────────────────────────────────────────────┘
```

### 4.4 装饰器设计

#### 4.4.1 @param_generator装饰器实现
```python
def param_generator(scope: str = "function", cache: bool = True, lazy: bool = True):
    """
    参数生成器装饰器
    
    参数:
        scope: 作用域，支持 function/class/module/session
        cache: 是否启用缓存
        lazy: 是否支持懒加载
    """
    
    def decorator(func: Callable) -> Callable:
        # 添加元数据，但不在此处创建ParamGenerator实例
        func._is_param_generator = True
        func._scope = scope
        func._cache_enabled = cache
        func._lazy_support = lazy
        func._decorator_args = {"scope": scope, "cache": cache, "lazy": lazy}
        
        # 保持函数原样，ParamGenerator实例将在注册时创建
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            return func(*args, **kwargs)
        
        return wrapper
    
    return decorator
```

#### 4.4.2 @with_dynamic_params装饰器实现
```python
def with_dynamic_params(**param_mapping: Callable):
    """
    使用动态参数装饰器
    
    参数:
        param_mapping: 参数名到生成器函数的映射
    """
    
    def decorator(test_func: Callable) -> Callable:
        # 验证所有生成器函数
        for param_name, generator_func in param_mapping.items():
            if not hasattr(generator_func, '_is_param_generator'):
                raise InvalidGeneratorError(
                    f"函数 {generator_func.__name__} 不是参数生成器，"
                    f"请使用 @param_generator 装饰器标记"
                )
        
        # 存储映射关系
        test_func._mapping = param_mapping
        test_func._requires = True
        
        # 生成器将在pytest_generate_tests钩子中注册
        # 在那里可以根据param_name创建ParamGenerator实例
        
        @functools.wraps(test_func)
        def wrapper(*args, **kwargs):
            return test_func(*args, **kwargs)
        
        return wrapper
    
    return decorator
```

### 4.5 错误处理设计

#### 4.5.1 异常类体系
```markdown
**DynamicParamError** (基础异常)
  ├── MissingParameterError
  │   - 属性: param_name, generator_name, available_params
  │   - 场景: 生成器依赖的参数在测试上下文中不存在
  │
  ├── CircularDependencyError
  │   - 属性: cycle
  │   - 场景: 检测到参数生成器之间的循环依赖
  │
  ├── InvalidGeneratorError
  │   - 属性: generator_func
  │   - 场景: 函数未使用@param_generator装饰器标记
  │
  ├── ConfigurationError
  │   - 属性: config_key, config_value, expected_type
  │   - 场景: 配置值无效或类型错误
  │
  └── ExecutionError
      - 属性: generator_name, exception, context
      - 场景: 参数生成器执行时抛出异常
```

#### 4.5.2 错误处理策略
```markdown
1. 验证阶段错误处理:
   - 在pytest_generate_tests钩子中捕获
   - 提供详细的错误信息，包括缺失的参数、可用参数列表
   - 抛出pytest异常，使测试收集失败

2. 执行阶段错误处理:
   - 在fixture中捕获生成器执行异常
   - 包装为ExecutionError，包含上下文信息
   - 抛出pytest异常，使测试执行失败

3. 配置错误处理:
   - 在系统初始化时捕获
   - 使用默认值并记录警告
   - 不影响系统基本功能
```

#### 5.1.2 懒加载fixture创建
```markdown
**函数**: create_lazy_fixture(param_name, generator_func)
**职责**: 为懒加载参数创建pytest fixture

**实现要点**:
1. 动态创建fixture函数
2. 在fixture中获取生成器实例
3. 收集执行上下文
4. 执行懒加载生成器
5. 处理缓存

**伪代码**:
function create_lazy_fixture(param_name, generator_func):
    @pytest.fixture
    def lazy_fixture(request):
        # 1. 获取生成器实例
        registry = GeneratorRegistry.get_instance()
        generator = registry.get_generator(param_name)
        
        if not generator:
            raise ValueError(f"未找到参数 {param_name} 的生成器")
        
        # 2. 收集上下文
        context = {}
        for dep in generator.dependencies:
            if hasattr(request, 'getfixturevalue'):
                try:
                    context[dep] = request.getfixturevalue(dep)
                except pytest.FixtureLookupError:
                    # 可能不是fixture，尝试从其他来源获取
                    pass
        
        # 3. 获取结果
        result = generator.get_result(context)
        
        # 4. 如果是懒加载结果，执行它
        if isinstance(result, LazyResult):
            return result.execute()
        else:
            return result
    
    return lazy_fixture
```
